## 1. Programme Overview

This Personal Learning Plan is designed to guide a learner through a structured, tutor-supported cohort programme in **Backend Development**. The programme emphasises real-world application, collaborative learning, and progressive skill-building — from core fundamentals to deploying production-grade systems.

### Core Philosophy

| Principle | Description |
|-----------|-------------|
| **Learn by Doing** | Hands-on projects precede passive reading |
| **Cohort Accountability** | Peers and tutors create shared responsibility |
| **Spiral Curriculum** | Concepts revisited at increasing depth |
| **Industry Alignment** | Every skill maps to real job requirements |
| **Reflective Practice** | Regular self-assessment and tutor feedback |

### Technology Stack Covered

```
Languages      → JavaScript (Node.js), Python, SQL
Frameworks     → Express.js, FastAPI / Flask
Databases      → PostgreSQL, MongoDB, Redis
DevOps         → Docker, Git/GitHub, CI/CD, Linux CLI
Cloud          → AWS / GCP basics, Deployment pipelines
Auth & Security→ JWT, OAuth2, HTTPS, Input Validation
Testing        → Jest, pytest, Postman, Integration tests
APIs           → REST
```

### 📗 Phase 1 — Foundations (Weeks 1–6)

**Objective:** Build a solid foundation in the tools and concepts all backend work relies on.

#### Week 1: Developer Environment & CLI

**Topics:**
- Terminal/Shell basics (navigation, file management, permissions)
- Installing and configuring development tools
- Introduction to VS Code and useful extensions
- Environment variables and `.env` files

**Deliverables:**
- [ ] Terminal quiz (10 CLI commands)
- [ ] Set up a complete local dev environment (documented in README)

**Tutor Session Focus:** Environment setup walkthrough + Q&A

---

#### Week 2: Version Control with Git & GitHub

**Topics:**
- Git fundamentals: init, add, commit, push, pull
- Branching strategies: feature branches, main/dev flow
- Pull Requests, code review basics
- `.gitignore`, merge conflicts

**Deliverables:**
- [ ] Create a GitHub repo with 5+ meaningful commits
- [ ] Open and merge a Pull Request with a peer review

**Tutor Session Focus:** Live git workflow demo + conflict resolution lab

---

#### Week 3: JavaScript Fundamentals for Backend

**Topics:**
- Variables, data types, functions, scope
- Asynchronous JS: callbacks, Promises, async/await
- ES6+ syntax: destructuring, spread, modules
- Error handling: try/catch, custom errors

**Deliverables:**
- [ ] Complete 10 JS exercises (async-focused)
- [ ] Build a simple CLI tool in Node.js

**Tutor Session Focus:** Async deep-dive + live debugging session

---

#### Week 4: Introduction to Node.js

**Topics:**
- Node.js runtime: how it works, event loop
- Built-in modules: `fs`, `path`, `http`, `os`
- NPM: installing packages, `package.json`, scripts
- Building a basic HTTP server without frameworks

**Deliverables:**
- [ ] Build a raw HTTP server that handles 3 routes
- [ ] NPM dependency management exercise

**Tutor Session Focus:** Node.js internals + event loop visualisation

---

#### Week 5: Express.js & REST API Basics

**Topics:**
- Express setup, routing, middleware
- HTTP methods: GET, POST, PUT, PATCH, DELETE
- Request/response cycle, status codes
- Body parsing, query params, route params

**Deliverables:**
- [ ] Build a CRUD API for a "Book Library" resource
- [ ] Document all endpoints in a README

**Tutor Session Focus:** Middleware deep-dive + Postman testing lab

#### Week 6: Phase 1 Review & Mini-Project

- Introduction to project documentation
- Phase 1 retrospective

**Deliverables:**

## 8. Assessment & Evaluation Framework

### Assessment Types

| Type | Frequency | Weight | Description |
|------|-----------|:------:|-------------|
| Weekly Exercises | Every week | 20% | Short coding tasks tied to that week's topic |
| Phase Projects | Phases 1–3 | 30% | Larger builds that demonstrate phase mastery |
| Peer Code Reviews | Weekly | 10% | Quality and consistency of feedback given |
| Mid-Programme Project | Week 12 | 15% | Full-stack auth + CRUD API |
| Capstone Project | Week 24 | 25% | End-to-end application, deployed and documented |

### Grading Rubric (Per Assignment)

| Criterion | Excellent (4) | Good (3) | Developing (2) | Needs Work (1) |
|-----------|:-------------:|:--------:|:--------------:|:--------------:|
| **Functionality** | All requirements met + extras | All requirements met | Most requirements met | Key requirements missing |
| **Code Quality** | Clean, DRY, well-named | Minor issues | Some repetition/unclear names | Hard to read/maintain |
| **Testing** | >80% coverage, edge cases | >60% coverage | Some tests | No tests |
| **Documentation** | Thorough README, inline comments | Good README | Basic README | No documentation |
| **Git Practice** | Meaningful commits, good PRs | Regular commits | Infrequent commits | One large commit |

### Feedback Loop

```
Submit assignment
       ↓
Automated checks run (linter, tests)
       ↓
Peer review (within 24 hrs)
       ↓
Tutor review + written feedback (within 48 hrs)
       ↓
Learner revision (optional, 48-hr window)
       ↓
Final grade logged
       ↓
1:1 debrief (bi-weekly)
```

## 13. Support Systems & Wellbeing

### Seeking Help: The 20-Minute Rule

> Before asking for help, spend **20 minutes** attempting to solve it yourself. Then, if stuck, ask — but document what you tried. This builds resilience and helps your helpers understand your thinking.

**Escalation path:**
1. Search official docs / MDN / Stack Overflow (20 min)
2. Ask in the cohort channel (tag `@facilitator`)
3. Raise in the next live session
4. Book 1:1 tutor office hours

### Managing Learning Fatigue

| Sign | Strategy |
|------|----------|
| Feeling overwhelmed by scope | Break into smaller tasks; review one concept at a time |
| Imposter syndrome | Look back at Week 1 code — you've already grown |
| Falling behind on assignments | Communicate with tutor immediately, plan a catch-up sprint |
| Burnout | Take a full day off; don't debug tired |
| Confusion persists after 2 sessions | Request a dedicated tutor review of that concept |

### Cohort Community Norms

- **Assume good intent** in all communications
- **Share blockers early** — the cohort grows together
- **Celebrate wins** — yours and your teammates'
- **No shame in asking** — every question benefits the group
- **Constructive code reviews only** — critique the code, not the person
